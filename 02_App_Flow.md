# App Flow Document
## VAMOS AI — Companion (Fan App) & Command (Staff Dashboard)

This document maps every persona's end-to-end journey through the product, screen by screen.

---

## 1. Surfaces Overview

```
VAMOS AI
├── VAMOS Companion (Fan-facing, mobile-first app)
│   ├── Onboarding & Language Setup
│   ├── Home / AI Concierge Chat
│   ├── Navigate (Navigation Agent)
│   ├── Crowd Awareness Layer (Crowd Agent, read-only for fans)
│   ├── Accessibility Mode (Accessibility Agent)
│   ├── Transport & Exit Planner (Transport Agent)
│   ├── Green Score (Sustainability Agent)
│   └── Profile & Settings
│
└── VAMOS Command (Staff/Organizer-facing web dashboard)
    ├── Login (role-based: Volunteer / Zone Staff / Control Room Operator)
    ├── Live Venue Overview (Crowd + Ops fused view)
    ├── AI Copilot Chat (Ops Intelligence Agent)
    ├── Incident Management
    ├── Decision Support Queue (approve/reject AI recommendations)
    ├── Dispatch & Volunteer Coordination
    └── Sustainability & Reporting Dashboard
```

## 2. Fan Journey — VAMOS Companion

### 2.1 Onboarding
1. **Splash → Language Detection**: App detects device language, GenAI confirms: *"Looks like you speak Portuguese — continue in Portuguese?"* (one tap to change).
2. **Ticket Link (optional)**: Fan scans/enters ticket QR → app pre-loads seat location, gate, match time.
3. **Accessibility Preferences (optional)**: Toggle: wheelchair/step-free routing, captions on, audio-description mode, sign-language avatar.
4. **Consent Screen**: Clear, plain-language notice — location used for navigation/crowd safety only, no facial recognition, anonymized crowd data.

### 2.2 Home Screen
- Central **AI Concierge chat bar** ("Ask me anything — in any language") — this is the primary entry point, not a menu-first design.
- Quick-action chips: `Find my seat` · `Nearest restroom` · `Food nearby` · `How do I get home?` · `Report an issue`
- Live **match countdown** + **venue status banner** (color-coded: green = normal flow, amber = busy, red = advisory in effect) generated by the Crowd Agent.

### 2.3 Navigation Flow (Navigation Agent)
1. Fan taps `Find my seat` or types "Where's Gate 14?"
2. **Orchestrator** routes to Navigation Agent, cross-checks with Crowd Agent for live zone status.
3. App shows: 2D map with generated route, ETA, and a one-line rationale: *"Routed via Concourse B — Concourse A is at 85% capacity."*
4. Voice mode: fan can say "read directions aloud" — Language Agent handles TTS in fan's language.
5. If a fan asks something ambiguous ("nearest bathroom"), Orchestrator merges Navigation + Accessibility (if step-free mode is on) + Crowd (shortest queue) into one generated answer.

### 2.4 Accessibility Mode
1. Fan enables from Settings or is auto-prompted at onboarding.
2. PA announcements appear as **live captions** in fan's language; sign-language avatar clip renders for key announcements (goal, safety, halftime).
3. Low-vision fans can tap `Describe surroundings` → Accessibility Agent generates a short spoken description of the immediate venue area and nearest facilities.
4. All routes generated for this fan automatically exclude stairs-only paths.

### 2.5 Crowd Awareness (read-only for fans)
- Fans see a simplified **3-color live status** per concourse/gate (not the full ops heatmap — that's staff-only) so they can self-select less busy routes.
- Push notification example (GenAI-drafted, human-approved template): *"Heads up — Gate 9 is very busy. Gate 11 is 4 minutes away and currently quiet."*

### 2.6 Transport & Exit Planner
1. **Pre-match**: fan opens `Getting there` → Transport Agent generates arrival recommendation (mode, time) based on predicted congestion.
2. **Post-match**: ~15 min before final whistle, app proactively surfaces: *"Based on your ticket zone, we recommend leaving via Exit C and taking the Blue Line — estimated 12 min shorter than Exit A right now."*
3. Fan can accept, get an alternate suggestion, or dismiss.

### 2.7 Green Score (Sustainability Agent)
1. Fan opts into tracking (transit mode used, reusable cup/bag scan, waste bin photo).
2. Agent generates a **Green Match Score** and a friendly one-line tip: *"You saved ~2.1kg CO2 by taking the train today — nice work."*
3. Venue-wide aggregate shown as a fun leaderboard/stat (opt-in, anonymized).

### 2.8 Report an Issue
1. Fan taps `Report an issue` → simple form (category + optional photo + location auto-tagged).
2. This feeds directly into **VAMOS Command's Incident Management**, summarized by the Ops Intelligence Agent for staff.

## 3. Staff/Organizer Journey — VAMOS Command

### 3.1 Login
- Role-based login (Volunteer / Zone Staff / Control Room Operator) via SSO.
- Volunteers get a lightweight **mobile view**; Control Room Operators get the **full desktop dashboard**.

### 3.2 Live Venue Overview (Control Room)
1. Central **venue map with live heatmap** (Crowd Agent) — full 5-level density per zone, refreshed every ~30s.
2. Side panel: **AI-generated Situation Summary**, refreshed continuously — e.g., *"Overall flow normal. Gate 14 trending toward high density in ~12 min if inflow continues. 2 open incidents, 1 medical, 1 lost child (resolved)."*
3. Weather, transit status, and match clock shown as context strip.

### 3.3 Decision Support Queue
1. When the **Decision Support Agent** detects an anomaly, a card appears: anomaly description, confidence score, and **2-3 ranked recommended actions** with plain-language rationale.
   - Example: *"Gate 14 density trending high (82% confidence). Recommended: (1) Open Gate 16 now, (2) Redirect incoming traffic via signage update, (3) Hold and monitor 5 more minutes."*
2. Operator taps **Approve**, **Modify**, or **Reject** — every decision is logged with operator ID + timestamp.
3. On approval, the system generates the exact instruction to send to affected volunteers/staff (see 3.5).

### 3.4 AI Copilot Chat (Ops Intelligence Agent)
- Any staff member can ask: *"Summarize everything happening at Gate 14 in the last 20 minutes"* or *"Draft a briefing for the 6pm shift change."*
- Agent generates a structured, sourced summary pulling from incident logs, crowd data, and chat history — never invents facts not present in the retrieved data.

### 3.5 Dispatch & Volunteer Coordination
1. Once an action is approved, the system **auto-generates a dispatch script** per affected volunteer/zone: *"Go to Gate 16. Open the gate. Direct incoming fans from Concourse B. ETA needed: 2 min."*
2. Sent via push notification to the relevant volunteers' mobile view.
3. Volunteer taps `Acknowledge` → `En route` → `Done`, updating the live status back to the control room.

### 3.6 Incident Management
- Kanban-style board: `New` → `Acknowledged` → `In Progress` → `Resolved`.
- Each incident auto-summarized on creation (from fan reports, volunteer reports, or system-detected anomalies).
- Full audit trail retained for post-event review.

### 3.7 Sustainability & Reporting Dashboard
- Aggregate venue sustainability metrics (transit mode split, waste diversion rate, Green Score participation).
- Post-match auto-generated **event report** (GenAI-drafted summary of operations, incidents, crowd patterns, sustainability stats) for organizer review/export.

## 4. Cross-Cutting Flow: How the Orchestrator Ties It Together

```
Fan or Staff Input (text/voice)
        │
        ▼
   Orchestrator (Intent Classification)
        │
        ├──► Single-agent path (e.g., pure translation request)
        │
        └──► Multi-agent path (e.g., "get me out of here safely, I'm in a wheelchair")
                 ├─ Navigation Agent   → step-free route
                 ├─ Crowd Agent        → avoid high-density zones
                 ├─ Accessibility Agent→ confirm step-free constraint
                 └─ Transport Agent    → best exit + transit combo
                          │
                          ▼
                 Merged, single coherent
                 generated response to user
```

Every response passes through a **guardrail check**: is this safety-critical? If yes → route to human approval queue (Decision Support flow) instead of auto-responding.
